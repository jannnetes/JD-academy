# JD Academy — освітня платформа

Повноцінна освітня платформа: каталог курсів, особисті кабінети учня / викладача / адміністратора,
покупки з автоматичним розрахунком комісії, live-заняття та домашні завдання.

## Стек
- **Frontend:** React 19 + Vite, React Router 7, Framer Motion
- **Backend:** Node.js + Express, JWT-авторизація (ролі), Prisma ORM
- **БД:** SQLite (локально) — на проді міняється на PostgreSQL в `server/prisma/schema.prisma`

---

## Як запустити локально

### 1. Перший запуск (один раз) — налаштувати бекенд і базу
```bash
npm run server:setup
```
Це встановить залежності сервера, створить БД і наповнить демо-даними.

### 2. Запустити все (фронт + бекенд) однією командою
```bash
npm run dev:all
```
- Сайт: **http://localhost:5173**
- API:  **http://localhost:4000**

> Можна запускати окремо: `npm run dev` (фронт) і `npm run dev:server` (бекенд).

## Що працює
- **Реєстрація/вхід** з вибором ролі (учень / викладач), JWT
- **Каталог** з фільтрами по галузі та пошуком
- **Сторінка курсу** з програмою, відгуками і прозорим розрахунком ціни (ціна + комісія)
- **Покупка курсу** (mock-оплата) → миттєвий доступ + запис у БД (Order + Enrollment)
- **Кабінет учня:** мої курси з прогресом, відмітка уроків, записи на live, історія покупок
- **Кабінет викладача:** аналітика (учні/дохід/рейтинг), створення та публікація курсів,
  планування live-занять, публікація ДЗ після заняття
- **Кабінет адміна:** статистика платформи, управління користувачами/ролями,
  фінанси (GMV + комісія), налаштування наценок (%)
- **Монетизація:** комісія рахується централізовано (`PlatformFeeConfig`), наценка зверху ціни,
  історичні замовлення не перераховуються

## Production

### Збірка фронтенду
```bash
npm run build
```
Бекенд автоматично віддає `dist/` як статику з SPA-fallback — тобто в проді все працює
з одного процесу:
```bash
npm run build
cd server && npm start   # http://localhost:4000 віддає і API, і сайт
```

### Перехід на PostgreSQL (прод)
1. У `server/prisma/schema.prisma` змінити `provider = "postgresql"`
2. У `server/.env` вказати `DATABASE_URL` від Postgres
3. `npm --prefix server run db:push && npm --prefix server run db:seed`
4. Змінити `JWT_SECRET` на надійний секрет

### Наступні кроки до повного продакшену
- Реальні платежі: Stripe Connect (міжнародно) або LiqPay/Fondy (Україна)
- Відеохостинг курсів: Mux / Cloudflare Stream; live: LiveKit / Daily.co
- Зберігання файлів: S3 / Cloudflare R2
- Деплой: Railway / Render (старт) → VPS/Kubernetes (масштаб)
